# CareTwin Home / 慧伴居

## One-line description

慧伴居是一套面向独居与半失能老人的非侵入式 AI 居家安全监护系统，通过多源传感器和行为异常识别，在老人无法主动求助时提前发现风险并联动家属与社区照护资源。

## Target competition context

This is a product-demo MVP for the Chunhui Cup innovation/startup competition. The system should be presented as a startup product, not merely a technical simulation.

## Target users

1. 独居老人家庭
2. 空巢老人子女
3. 社区居家养老服务中心
4. 养老公寓 / 护理机构
5. 地方智慧养老项目

## Core pain points

1. 老人跌倒、夜间滞留、忘记服药等风险无法及时发现。
2. 老人不一定会主动按下求救按钮。
3. 摄像头监控侵犯隐私，老人和家庭接受度低。
4. 人工巡查成本高，社区护理人员无法实时覆盖所有家庭。
5. 子女远程照护缺乏可信、可解释的信息。

## Product value proposition

CareTwin Home provides:

1. Non-camera elderly-care monitoring.
2. Individual routine learning.
3. Risk event detection.
4. Explainable alerts.
5. Smart-home response.
6. Family and community-care notification.
7. Daily care summary.

## MVP scope

The MVP must simulate:

- A smart elderly-care apartment.
- Multiple non-camera sensors.
- Elderly movement and activity.
- AI risk scoring.
- Smart-home actions.
- Family alert.

## Out of scope for MVP

- Real hardware connection.
- Medical diagnosis.
- Emergency-service integration.
- Payment or user account system.
- Production-grade deployment.
- Long-term ML model training.

## Sensor types

Use simulated versions of:

1. Bed pressure sensor
2. PIR / mmWave presence sensor
3. Door contact sensor
4. Smart plug
5. Medication box open sensor
6. Smoke / gas sensor
7. Ambient temperature and humidity sensor

## Core rooms

1. Bedroom
2. Hallway
3. Bathroom
4. Kitchen
5. Living room
6. Entrance

## Core abnormal events

1. Night bathroom prolonged stay
2. Long static inactivity
3. Missed medication
4. Kitchen gas/smoke risk

## Demo success definition

A judge should understand within 3 minutes:

1. Who the user is.
2. What risk is being detected.
3. Why camera-free monitoring matters.
4. How AI risk scoring works.
5. What action the system takes.
6. Why this can become a real product.
